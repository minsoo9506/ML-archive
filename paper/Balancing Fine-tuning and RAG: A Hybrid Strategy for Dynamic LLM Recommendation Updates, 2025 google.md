# Introduction
- LLM 기반 추천 시스템에서 사용자 관심사와 콘텐츠가 동적으로 변화하는 상황에서 추천 품질을 유지하는 문제를 다룸
- 초기 파인튜닝은 강력한 성능을 제공하지만, 실시간 행동 변화를 포착하지 못하는 "정적 스냅샷" 문제가 존재
- 월간 top-5 후속 관심사 변화를 Jaccard Similarity로 측정한 결과, 평균 0.17(분산 0.07)로 매월 상당한 변동이 확인됨
- 핵심 가설: 정적 파인튜닝된 LLM만으로는 추천 품질 유지가 불충분하며, 주기적 파인튜닝과 빈번한 RAG 업데이트를 결합한 하이브리드 전략이 필요
- 이를 극복하기 위해 fine-tuning & RAG를 결합한 하이브리드 업데이트 전략 제안

# Method
## Preliminary
- 사용자의 시청 이력을 사전 정의된 아이템 클러스터 k길이 시퀀스로 표현
  - 아이템의 메타, 콘텐츠 데이터로 topically coherent 한 클러스터
- 입력: (c₁, c₂) 클러스터 시퀀스 → 출력: LLM 예측된 후속 클러스터 c_next (successor interest)
  - 수십억 사용자 규모의 계산 제약으로 인해, 가능한 모든 클러스터 시퀀스를 사전에 예측하여 룩업 테이블로 저장
  - online serving 시 유저 history 시쿼스에 매칭된 예측값 return
 
## Fine-tune
- 학습 샘플: 사용자 상호작용 로그에서 추출한 [(c₁, c₂), c_next] 쌍
- 업데이트 주기: 높은 계산 비용으로 인해 월간 업데이트
- 품질 게이트 적용:
  - Exact match rate > 90%
  - Test set recall > 1.5%
- 장점: 모델 행동을 적응시키고 특정 태스크에 맞춤 성능 제공
- 단점: 높은 비용, 복잡성, 오버피팅 위험

## RAG
- 재학습 없이 최신 트렌드 데이터를 프롬프트에 주입하여 추론 → 주 여러 회, 심지어 매일 갱신 가능 (저비용)
- bulk inference 단계에서 모든 클러스터 쌍 (c₁, c₂)에 대해 "다음 관심사를 예측해줘"라고 LLM에 요청할 때, 최신 데이터 기반 힌트를 프롬프트에 함께 제공
- 설계 차원 두 가지를 평가:
  - **Granularity** (프롬프트 생성 단위):
    - Instance-level (채택): 각 클러스터 쌍마다 개별적으로 최빈 후속 클러스터를 찾아 맞춤 프롬프트 생성
      - 예: 최근 데이터에서 (c₁, c₂) 다음에 c₃: 10번, c₄: 8번, c₅: 2번 등장했다면 → 가장 빈번한 c₃을 프롬프트에 포함하여 LLM에게 "최근에는 c₃이 가장 많이 나왔어"라는 힌트 제공
    - Global-level (기각): 모든 쌍에 범용 프롬프트 사용 → 노이즈 과다
  - **Retrieval Similarity** (데이터 검색 기준):
    - Frequency-based (채택): 쿼리 클러스터 쌍에 대해 최고 빈도 데이터 포인트 검색
    - Trend-based (기각): 급등하는 트렌드 쌍 → 성능 저하
  - retrieved cluster 의 갯수는 computational cost 를 고려해서 top1 이면 충분하다고 한다.

## Data Retrieval
- 대규모 비디오 플랫폼(YouTube) 사용자 상호작용 이력에서 데이터 추출
- 중복 제거된 시퀀스(최소 2개 비디오)에서 연속 3개 클러스터 튜플 (c₁, c₂, c_next) 구성
- 각 쌍에 대한 후속 클러스터 (c_next) 출현 빈도 계산


# Results and Evaluation
## Offline Evaluation
- Hit rate (예측 클러스터가 실제 시퀀스에 등장하는 비율): 2일마다 RAG 업데이트한 매핑이 고정 베이스라인 대비 우수
- RAG 출력의 7.8%만이 비-RAG 출력과 일치 (동일 프롬프트 반복 시 37.5% 일치) → RAG가 생성 결과에 실질적 영향을 미침
- Top-k 클러스터 중복은 시간이 지남에 따라 크게 감소 → 빈번한 갱신 주기의 필요

## Live Experiment
- 플랫폼: 수십억 사용자 규모의 숏폼 비디오 시스템 (YouTube)
- 기본 LLM: Gemini 1.5
- RAG 업데이트: 실험 1일차, 4일차에 적용
- 결과:
  - Satisfied User Outcomes: **+0.11%** [95% CI: 0.00%-0.21%]
  - Satisfaction Rate: **+0.25%** [95% CI: 0.01%-0.48%]
  - Dissatisfaction Rate: **-0.05%** [95% CI: -0.08% ~ -0.01%]
  - Negative Interaction: **-0.04%** [95% CI: -0.08% ~ -0.01%]
- RAG 업데이트 적용 직후(1일차, 4일차) 만족도 지표가 눈에 띄게 상승

# Conclusion
- 월간 파인튜닝 + 주 단위 이하 RAG 업데이트의 하이브리드 접근법이 동적 사용자 관심사 적응을 위한 강건하고 비용 효율적인 솔루션임을 입증
- 파인튜닝만으로는 빠르게 변화하는 도메인에서 불충분
- RAG는 재학습 없이 빈번하고 저비용의 적응을 가능하게 함
- 하이브리드 전략이 프로덕션 규모에서 사용자 만족도 지표를 통계적으로 유의미하게 개선
- 향후 연구: 감지된 관심사 변동률에 기반한 적응적 갱신 주기 탐색
